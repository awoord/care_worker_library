1. プロジェクト概要
目的: 介護福祉士国家試験の頻出250語を網羅した、印刷用（A4）単語帳データの構築。

主要技術: Python, Jinja2, JSON, HTML/CSS.

成果物: 単語帳用データは `books/extracted_words.json`（既定入力）、従来の `books/words.json` は `generate_word_book.py --input` で指定可能。表示は `books/generate_word_book.py` と `books/word_book.j2` 等。

2. データ構造定義 (Strict)
各要素は、以下のキーを持つこと（`books/words.json` 利用時は `pos` 必須。`extracted_words.json` も同じキー構成に揃える）。

word: 見出し語（日本語）

count: 過去問出現回数（数値）

meaning: 簡潔な日本語の意味（JLPT N3 想定の学習者にも通じる平易な表現を優先）

english: 英訳。原則は平易な単語を1語。1語だと意味が誤解されるときのみ2語（スラッシュ区切り可）。

pos: 品詞（名・動・形・副のいずれか）。単語帳 HTML で `[名]` 形式表示に使用。

example: 過去問から抽出した例文

source: 出典（形式: 回次-問番号 例: 38-3、36-98）

3. ハルシネーション防止策 (厳守)
抽出優先: 例文は必ず `kakomon/kaigo_kakomon_32.txt` 〜 `kaigo_kakomon_38.txt` から、見出し語を含む箇所を一字一句変えずに抽出すること。一文にこだわらず、意味が通る範囲で複数文・選択肢の行から取ってよい。問題文を優先し、平易な日本語の箇所を選ぶこと。

過去問は最新回（番号の大きいファイル）を優先して当たること。別語の例文と出典が重複してもよい。

過去問ファイルの例: `kaigo_kakomon_32.txt` と `kaigo_kakomon_38.txt` の両方に該当があれば、`kaigo_kakomon_38.txt` を先に検索する。

捏造禁止: 過去問内に該当語が見当たらない場合のみ、AIが「試験レベルに即した自然な例文」を作成し、source を "AI-Generated" と記述すること。

検証義務: データを抽出・生成した際、必ずソースファイル（txt）のどの部分を参照したかを内部的にクロスチェックすること。

4. 開発フェーズ・プロトコル
作業は以下のフェーズに分け、一括ではなくフェーズごとにユーザーの承認を得て進めること。

Phase 1: 環境整備: 既存の .py や .j2 のパス問題を修正し、ブラウザ表示が完璧に動く状態にする。

Phase 2: 逐次データ抽出: 約270語を一気に処理せず、20語ずつ `extracted_words.json`（または `words.json`）を更新し、進捗を報告すること。

Phase 3: レイアウト最適化: A4印刷時に1ページ10語が収まり、デザイン（CSS）が崩れないよう調整する。

5. 禁止事項
「完璧」「最強」といった装飾的な言葉を回答に使用しないこと。

ユーザーの許可なく、既存の JSON の必須キー構成（word, count, meaning, english, example, source および単語帳用の pos）を勝手に外さないこと。

CSSにおいて、単語カードがページを跨いで分割される（不自然な改ページ）ことを許容しない。


## 0. ターゲット学習者情報
* **背景**: 外国人受験生（フィリピン人等のEPN/技能実習生を想定）
* **目的**: 介護福祉士国家試験の合格、および現場での実務コミュニケーション
* **ニーズ**: 
    - 全ての漢字に「正確な読み（ルビ）」が必要。
    - 日本語の能力はJLPTのN3レベル。
    - 日本語の「意味」は、できるだけ平易な表現（やさしい日本語）を優先。
